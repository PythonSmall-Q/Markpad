# 安全策略

| 版本 | 支持状态 |
| --- | --- |

## 报告漏洞

如果你发现了安全漏洞，请**不要**公开提交 Issue。

请通过以下方式报告：

1. 发送邮件到安全团队（如果有）
2. 或者创建一个私有的 Security Advisory

### 报告内容应包括

- 漏洞类型
- 受影响的版本
- 重现步骤
- 潜在影响
- 可能的修复方案（如果有）

### 响应流程

1. 我们会在 48 小时内确认收到报告
2. 我们会评估漏洞并制定修复计划
3. 我们会在修复完成后通知你
4. 修复发布后，我们会公开致谢

## 安全最佳实践

作为用户，请：

- 保持应用程序更新到最新版本
- 不要从不可信的来源下载文件
- 注意可疑的链接和内容
- 定期备份重要文档

作为开发者，我们：

- 定期更新依赖
- 进行安全审查
- 实施最小权限原则
- 使用安全的 IPC 通信

感谢你帮助保护 Markpad 及其用户！
